# Luther Bible MCP Server

A semantic search MCP server for the German Luther Bible (1545, modern spelling), with King James Version and New Heart English Bible included. Backed by `intfloat/multilingual-e5-small` embeddings and ChromaDB. Plugs directly into Claude Desktop.

Cross-lingual search works out of the box — query in English to find German verses, or vice versa. No API key required.

---

## Project structure

```
luther_mcp/
  constants.py         # shared book names, translation metadata
  server.py            # MCP server implementation (stdio + SSE)
  indexer.py           # one-time indexer (builds ChromaDB from SQLite)
  downloader.py        # downloads pre-built ChromaDB from GitHub Releases
  __main__.py          # entry point: python -m luther_mcp [serve|index|download|--sse]
hf_space/              # HuggingFace Spaces deployment (Docker)
tests/
  test_server.py       # unit tests (mocked, no external calls)
bible_chroma_db/       # generated by indexer or download (gitignored)
```

## Setup

### 1. Install dependencies

```bash
cd C:\Projects\luther-mcp
pip install -r requirements.txt
```

### 2. Get the ChromaDB (choose one)

#### Option A — Download pre-built (~700 MB, free, ~1 minute)

```bash
set CHROMA_PATH=C:\Projects\luther-mcp\bible_chroma_db
python -m luther_mcp download
```

#### Option B — Build from scratch (~10 minutes, no API key needed)

Clone the Bible data:
```bash
cd C:\Projects
git clone --depth 1 --filter=blob:none --sparse https://github.com/scrollmapper/bible_databases
cd bible_databases
git sparse-checkout set "formats/sqlite/GerBoLut.db" "formats/sqlite/KJV.db" "formats/sqlite/NHEB.db"
git checkout
```

Run the indexer:
```bash
cd C:\Projects\luther-mcp
python -m luther_mcp index
```

To re-index from scratch: `python -m luther_mcp index --force`

## Add to Claude Desktop (local)

Edit `claude_desktop_config.json` (at `%APPDATA%\Claude\claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "Luther Bible": {
      "command": "C:\\Python313\\python.exe",
      "args": ["-m", "luther_mcp"],
      "cwd": "C:\\Projects\\luther-mcp",
      "env": {
        "CHROMA_PATH": "C:\\Projects\\luther-mcp\\bible_chroma_db",
        "PYTHONPATH": "C:\\Projects\\luther-mcp"
      }
    }
  }
}
```

`PYTHONPATH` is required — Claude Desktop does not add `cwd` to Python's module search path automatically.

Restart Claude Desktop. The server appears under the MCP tools panel.

## Available tools

### `search_bible`
Semantic search — finds verses by meaning, not keyword.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `query` | string | required | Query in any language |
| `translation` | string | `"GerBoLut"` | `"GerBoLut"` / `"KJV"` / `"web"` / `"all"` |
| `n_results` | integer | `10` | Results to return (max 50) |
| `testament` | string | — | `"OT"` or `"NT"` to filter |

### `get_verse`
Exact reference lookup by book/chapter/verse.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `book` | string | required | German or English book name |
| `chapter` | integer | required | Chapter number |
| `verse` | integer | required | Verse number |
| `translation` | string | `"GerBoLut"` | `"GerBoLut"` / `"KJV"` / `"web"` / `"all"` |

### `list_translations`
Returns available translations with language, description, and verse count.

## Example queries

### German queries
- `"Gottes Liebe zur Welt"` → finds John 3:16 and similar verses
- `"Licht in der Dunkelheit"` → Gospel of John prologue, Genesis creation
- `"Vergebung der Sünden"` → forgiveness passages
- `"Seid nicht ängstlich"` → "Do not be afraid" passages

### English queries (cross-lingual — returns German Luther Bible verses)
- `"God so loved the world"` → Johannes 3:16
- `"blessed are the peacemakers"` → Matthäus 5:9
- `"love your enemies"` → Matthäus 5:44
- `"the Lord is my shepherd"` → Psalmen 23:1

### Exact lookup
- Luther Bible: `get_verse("Johannes", 3, 16)`
- All translations: `get_verse("John", 3, 16, translation="all")`

## Deploying to HuggingFace Spaces (free hosting)

The `hf_space/` directory contains everything needed to deploy a public, always-on hosted version.

### 1. Create the Space

Go to [huggingface.co/new-space](https://huggingface.co/new-space):
- Owner: `happyherp`
- Space name: `luther-bible`
- SDK: **Docker**
- Visibility: Public

### 2. Push the deployment files

```bash
git clone https://huggingface.co/spaces/happyherp/luther-bible hf-deploy
cp hf_space/README.md hf-deploy/README.md
cp hf_space/Dockerfile hf-deploy/Dockerfile
cp hf_space/requirements.txt hf-deploy/requirements.txt
cp -r luther_mcp hf-deploy/luther_mcp
cd hf-deploy && git add -A && git commit -m "initial deployment" && git push
```

### 3. Index the Bible on the Space

Once the Space is running, open its terminal and run:

```bash
# Upload the SQLite files to /data/bible_databases/ first, then:
python -m luther_mcp index
```

The index will be stored in `/data/bible_chroma_db/` (persistent storage, survives restarts).

### 4. Connect from Claude Desktop

```json
{
  "mcpServers": {
    "Luther Bible": {
      "url": "https://happyherp-luther-bible.hf.space/sse"
    }
  }
}
```

**Note on cold starts:** Free CPU spaces may sleep after ~48h of no traffic. The first request after waking takes ~10–15 seconds (model reload). Subsequent requests are fast (~50ms).

## Publishing a release (maintainers)

New users can skip re-indexing by downloading the pre-built ChromaDB via
`python -m luther_mcp download`. To publish that artifact to GitHub Releases:

### 1. Create a GitHub Personal Access Token

Go to **https://github.com/settings/tokens/new** and create a token with
**Contents** scope (read + write). Copy it.

### 2. Run the release script

```bash
set GITHUB_TOKEN=ghp_your_token_here
python scripts/create_release.py
```

The script will:
1. Compress `bible_chroma_db/` → `bible_chroma_db.tar.gz`
2. Create a GitHub release tagged with the current version
3. Upload the archive as a release asset
4. Delete the local archive when done

### 3. Releasing a new version

Bump `__version__` in `luther_mcp/__init__.py`, commit, then re-run the script.
The release tag and download URL are derived automatically — no other files need editing.
